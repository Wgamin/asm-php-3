 183: Laravel.
 184: Laravel là một framework PHP mã nguồn mở, được phát triển theo mô hình MVC (Model - View - Controller), giúp xây dựng các ứng dụng web nhanh chóng, bảo mật và dễ bảo trì. Trong dự án Website bán trang sức SORA, Laravel được sử dụng để phát triển phần backend, xử lý toàn bộ logic nghiệp vụ như quản lý sản phẩm, đơn hàng, người dùng và dữ liệu hệ thống.
 185: Laravel cung cấp nhiều công cụ mạnh mẽ như hệ thống routing linh hoạt, migration và seeder giúp quản lý cơ sở dữ liệu hiệu quả. Ngoài ra, Laravel tích hợp Eloquent ORM, cho phép truy vấn dữ liệu theo hướng đối tượng, giúp giảm thiểu việc viết các câu lệnh SQL phức tạp.
 186: Bên cạnh đó, Laravel còn hỗ trợ các tính năng bảo mật như CSRF Token, xác thực người dùng, mã hóa mật khẩu và middleware để kiểm soát luồng xử lý. Framework cũng hỗ trợ xây dựng API (RESTful), giúp kết nối hiệu quả với frontend Vue.js.
 187: Visual Studio Code (VS Code)
 188: Visual Studio Code (VS Code) là trình soạn thảo mã nguồn miễn phí, được phát triển bởi Microsoft, được sử dụng trong dự án Website bán trang sức SORA để lập trình và quản lý mã nguồn.
 189: VS Code hỗ trợ nhiều ngôn ngữ lập trình thông qua các extension, đặc biệt phù hợp với các công nghệ sử dụng trong dự án như PHP, JavaScript và Vue.js. Công cụ này cung cấp các tính năng như tự động hoàn thành mã (IntelliSense), debug, tìm kiếm, quản lý Git và hỗ trợ làm việc nhóm hiệu quả.
 190: Việc sử dụng VS Code giúp nhóm phát triển tăng năng suất làm việc, dễ dàng chỉnh sửa, kiểm tra và triển khai mã nguồn trong quá trình xây dựng hệ thống.
 191: Vue.
 192: Vue.js là một framework JavaScript mã nguồn mở được sử dụng để xây dựng giao diện người dùng (frontend) cho hệ thống SORA. Vue giúp tạo ra các giao diện web hiện đại, mượt mà và có khả năng tương tác cao.
 193: Vue cho phép kết hợp HTML, CSS và JavaScript một cách linh hoạt. Với cơ chế data binding và Virtual DOM, Vue giúp tối ưu hiệu suất và cập nhật giao diện nhanh chóng khi dữ liệu thay đổi.
 194: Một số tính năng nổi bật được sử dụng trong dự án gồm:
 195: Component: Chia nhỏ giao diện thành các phần độc lập, dễ quản lý và tái sử dụng.
 196: Directive (v-if, v-for, v-model…): Hỗ trợ xử lý logic trực tiếp trong giao diện.
 197: Vue Router: Điều hướng giữa các trang trong ứng dụng.
 198: State management: Quản lý dữ liệu dùng chung giữa các component.
 199: Nhờ đó, Vue giúp nâng cao trải nghiệm người dùng trong quá trình mua sắm trang sức trực tuyến.
 200: GitHub.
 201: GitHub là nền tảng lưu trữ mã nguồn trực tuyến, sử dụng hệ thống quản lý phiên bản Git, được nhóm sử dụng để quản lý và phát triển dự án Website bán trang sức SORA.
 202: GitHub cho phép:
 203: Lưu trữ và quản lý mã nguồn.

 205: Theo dõi lịch sử thay đổi của dự án.
 206: Trong quá trình phát triển, nhóm sử dụng các thao tác cơ bản của Git như:
 207: Commit: Lưu thay đổi
 208: Push: Đẩy code lên repository
 209: Pull: Cập nhật code mới
 210: Merge: Gộp code từ các nhánh
 211: Việc sử dụng GitHub giúp đảm bảo tính nhất quán của mã nguồn, hỗ trợ làm việc nhóm và nâng cao chất lượng dự án.

 601: Phần trăm test case:
 602: Nhận xét chung:
 603: - Tỷ lệ Pass ở mức tương đối cao (≈ 79.4%), cho thấy phần lớn các chức năng hoạt động đúng theo yêu cầu.
 604: - Tuy nhiên, với 151 test case bị Fail (chiếm hơn 1/5 tổng số) (≈ 20.6%), vẫn còn một số vấn đề cần được xem xét và sửa lỗi kịp thời.
 605: - Nhóm cần ưu tiên xử lý các test case bị Fail, sau đó thực hiện kiểm thử lại (retest) để xác minh tính ổn định của hệ thống.
 606: -  Đánh giá kết quả kiểm thử sau khi fix bug: giảm Test Case FAIL xuống 5.2% và nâng Test Case PASS lên 94.8%.
 607: Sau khi fix bug:
 608: ĐÓNG GÓI VÀ TRIỂN KHAI
 609: KẾT LUẬN
 610: Kết quả đạt được:
 611: Với 3 tháng làm việc, nhóm Arise đã đã khảo sát và xây dựng được mô hình cấu trúc hệ thống luồng liên kết các chức năng của dự án.
 612: Thuận lợi:
 613: Có sự hỗ trợ của công nghệ tiên tiến.
 614: Có nhiều nguồn tài liệu và sự hỗ trợ từ giảng viên hướng dẫn để thực hiện dự án.
 615: Có thời gian đủ dài để phân chia thời gian làm việc, khảo sát, xây dựng và hoàn thiện các khía cạnh nghiệp vụ công việc của dự án.
 616: Khó khăn:
 617: Không ứng dụng hoàn toàn được các chức năng của công nghệ.
 618: Các nguồn tài liệu xem đọc thường bị bỏ qua nội dung trọng yếu để áp dụng thường phải xem lại lần nữa tốn thời gian quay lại để tham khảo tài liệu.
 619: Thời gian không được tối ưu hóa trong việc phân chia để phù hợp cho việc  hoàn thiện các nghiệp vụ chức năng của hệ thống chức năng dự án.
